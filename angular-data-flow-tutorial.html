Data Flow in the Same Component

This is for sharing data between the view (HTML file) and the component (TS file)

Binding

One-Way Binding

A common and easy method for transferring data between a component's view (HTML) and component (TS) is with one-way binding. This includes interpolation {{}}, property binding [], and event binding ().

Interpolation {{}}

Interpolation {{}} passes data one way, from the component (TS) file to the view (HTML) file. You cannot put expressions into the {{}}, only fields or variables. In addition, you can optionally include a pipe | to transform the field or variable in some way. Property binding [] also passes data from the component (TS) file to the view (HTML); the difference is that interpolation {{}} displays the string value of a variable (after being transformed by a pipe |, if applicable) or a component's field, whereas property binding [] sets the DOM property of an element to be the value of a field or variable.

Example of Interpolation {{}}:
const pageTitle: string = "my site";
let numVisitors = 6;
<p>Welcome to {{ pageTitle | uppercase }}!</p>
<br>
<p>Number of visitors: {{ numVisitors }}</p>
Welcome to My Site!
Number of visitors: 6
Property Binding []

Property binding [] passes data one way, from the component (TS) file to the view (HTML) file. This is the same as interpolation {{}}; the difference between property binding [] and interpolation {{}} is that interpolation {{}} displays the string value of a variable (after being transformed by a pipe |, if applicable) or a component's field, whereas property binding [] sets the DOM property of an element to be the value of a field or variable.

@ViewChild() (To Get DOM Element Data)

Although the @ViewChild() decorator is usually used to pass data between parent components and child components, it can also be used by a single component to gain access to the interior properties of an element in the DOM.
